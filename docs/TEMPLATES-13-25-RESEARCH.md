## Template 13: Y2K Retro

### Design Concept
Nostalgic 90s/2000s aesthetic with chrome effects, gradients, and playful elements.

### Key Visual Elements
- **Colors**: Metallics (silver, chrome), neon blues, purples, pinks
- **Typography**: Geometric sans-serifs, variable fonts, outline text
- **Effects**: Chrome/metallic gradients, drop shadows, glow effects
- **Shapes**: Rounded rectangles, pills, blobs, geometric patterns
- **Imagery**: Pixel art style, retro icons, star patterns

### Sections Structure
1. Hero with chrome text effect and gradient background
2. About with nostalgic badges/stickers aesthetic
3. Projects grid with metallic cards
4. Skills as retro badges
5. Timeline with Y2K aesthetic
6. Contact with gradient buttons

### Technical Features
- CSS gradients for chrome effect
- Text shadow for depth
- Border radius for soft shapes
- Animated gradient backgrounds
- Retro color palette CSS variables

### Typography
- **Primary**: "Space Grotesk" or "Orbitron" (geometric, futuristic)
- **Accent**: Variable width for dramatic headers

### Inspiration Sources
- Early 2000s websites (Geocities aesthetic)
- Windows XP/Vista design language
- iPod/early iPhone era
- Retro gaming interfaces

### Key Visual Elements
- **Grid**: Strict grid system
- **Typography**: Sans-serif, hierarchical

- **Whitespace**: Generous, intentional

### Sections Structure
1. Hero with grid-aligned elements
2. Modular grid-based sections
3. Projects in perfect grid
4. Systematic information architecture

### Technical Features

- Mathematical spacing system (8px grid)
- Limited color palette
- Precise typography scale
- Clean, functional animations

### Typography
- **Primary**: "Helvetica Neue" or "Inter" (neutral sans)

### Inspiration Sources
- Swiss design posters
- Vitsoe website
- Massimo Vignelli's work

## Common Technical Requirements

All templates will include:
- **Responsive Design**: Mobile-first, breakpoints at 768px, 1024px
- **Accessibility**: WCAG 2.1 AA compliant, keyboard navigation, screen reader support
- **Performance**: <2s load time, optimized assets
- **Dark Mode**: Where appropriate for the style
- **Forms**: Contact forms with validation
- **SEO**: Meta tags, semantic HTML
- **Documentation**: README with customization guide, config.json

## Code Structure Pattern

Each template folder will contain:
```
/templates/XX-template-name/
  ├── index.html        (semantic HTML5)
  ├── styles.css        (CSS with variables, mobile-first)
  ├── script.js         (vanilla JS, progressive enhancement)
  ├── config.json       (template metadata)
  └── README.md         (full documentation)
```

## CSS Architecture

All templates use:
- CSS Custom Properties for theming
- Mobile-first responsive design
- BEM-like naming for clarity
- Logical property ordering
- No preprocessors (pure CSS)

## JavaScript Approach

All templates use:
- Vanilla JavaScript (no frameworks)
- Progressive enhancement
- Event delegation where appropriate
- Intersection Observer for scroll effects
- Form validation and submission handling
